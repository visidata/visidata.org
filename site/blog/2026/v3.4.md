## [v3.4](https://github.com/saulpw/visidata/releases/tag/v3.4) (2026-06-30)

It's been the better part of a year since v3.3, and v3.4 is a big one: new features across the board, a thorough pass on markup rendering, and dozens of bugfixes. Highlights include:

- **Search highlighting** (`hlsearch`): every match for `/` and `g/` is now highlighted in place; toggle with `options.highlight_search`. ([#2861](https://github.com/saulpw/visidata/issues/2861))
- **Human-readable keys everywhere**: the "prettykeys" overhaul means all bindings display with readable names (`Ctrl+Up`, `Alt+d`, …) in help, menus, and status. ([#2594](https://github.com/saulpw/visidata/issues/2594))
- **Smarter `-f`**: filetype now separated between input and output, with `-f` applying to the input filetype, and with `-of` for the output filetype, which supports compound `format+compression` types like `json+gz`; an explicit `-f` also overrides URL-scheme dispatch. ([#1242](https://github.com/saulpw/visidata/issues/1242), [#573](https://github.com/saulpw/visidata/issues/573), [#985](https://github.com/saulpw/visidata/issues/985), [#3126](https://github.com/saulpw/visidata/issues/3126))
- **Define your own commands** in-session: `define-command <longname> <execstr>`, with `currow.<ColName>` available to read the cursor row's values. ([#655](https://github.com/saulpw/visidata/issues/655))
- **Save edited options** straight to your config with `z Ctrl+S`. ([#2206](https://github.com/saulpw/visidata/issues/2206))
- **Split-pane file preview** on the DirSheet. ([#3024](https://github.com/saulpw/visidata/issues/3024))
- **Customizable cell-editing keys** via `vd.editCellBindings`. ([#2986](https://github.com/saulpw/visidata/issues/2986))
- **Jump to any sheet by name** with `jump-sheet` (with tab-completion).
- **Change displayed float precision** with `setcol-precision` (`Alt+-`/`Alt++`/`g%`).
- **Drop into an interactive `$SHELL`** with `exec-shell`.
- **Aggregation bar** showing aggregate totals. ([#2209](https://github.com/saulpw/visidata/issues/2209), [#2995](https://github.com/saulpw/visidata/issues/2995))
- **Fuzzy input-history palette**. ([#2684](https://github.com/saulpw/visidata/issues/2684))
- **Macros** now support keystroke bindings and input parameters. ([#2784](https://github.com/saulpw/visidata/issues/2784), [#2785](https://github.com/saulpw/visidata/issues/2785))
- **Smarter SQLite `exec-sql`**: named queries, types, and edit-sql. ([#2136](https://github.com/saulpw/visidata/issues/2136), [#3020](https://github.com/saulpw/visidata/issues/3020))
- **GeoParquet** support: decode WKB geometry and plot it.
- **vdsql improvements**: multi-schema browsing ([#2027](https://github.com/saulpw/visidata/issues/2027)) and SQL shown in the sidebar ([#2200](https://github.com/saulpw/visidata/issues/2200)).
- **S3** recursive directory listing and anonymous-mode access.
- **Better HTML tables**: handle interspersed th/td and rowspan. ([#1346](https://github.com/saulpw/visidata/issues/1346), [#1308](https://github.com/saulpw/visidata/issues/1308))
- **CSV** now skips blank lines. ([#3085](https://github.com/saulpw/visidata/issues/3085))
- New **adwaita** light and dark themes. ([#3006](https://github.com/saulpw/visidata/issues/3006))
- An embedded **ptipython** REPL. ([#2736](https://github.com/saulpw/visidata/issues/2736))
- A **command server** for scripting VisiData.
- Experimental: **vdaw**, a transcript-based audio editor.
- Experimental: an **llm** plugin to analyze columns with Anthropic Claude. ([#2947](https://github.com/saulpw/visidata/issues/2947))
- Experimental: **search_geo**, bbox-overlap helpers for STAC catalogs.

Thanks to @midichef for a huge share of the bugfixes and improvements in this release, and to everyone who contributed (and for the first time including @saulbert, who helped ship several features we wouldn't have done otherwise).
